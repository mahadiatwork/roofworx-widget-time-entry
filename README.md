# Roof Worx Time Entry Widget

Zoho CRM embedded widget for field crew to log time against a job directly from the CRM job record.

## Quick start

```bash
npm install
npm run dev          # local dev at http://localhost:5173 (mock data, no Zoho SDK)
npm run widget:build # build + sync to app/ for Zoho packaging
```

## Zoho packaging

1. Install Zoho CLI: `npm install -g zoho-extension-toolkit`
2. Run `npm run widget:build`
3. `zet validate` then `zet pack` to produce a zip
4. Upload via **Setup → Developer Space → Extensions**

Or use `zet run` to serve locally at `http://127.0.0.1:5000/app/widget.html`.

## Configuration checklist

Before deploying, update **[src/config/zohoSchema.js](src/config/zohoSchema.js)** with your org's real API names:

| Config key | Purpose | Default |
|---|---|---|
| `timeEntryModule` | Custom Time Entry module API name | `Time_Entry` |
| `jobsModule` | Jobs / Deals module API name | `Deals` |
| `timeEntryFields.job` | Lookup to job record | `Job` |
| `timeEntryFields.date` | Work date | `Date` |
| `timeEntryFields.startTime` | Start time | `Start_Time` |
| `timeEntryFields.endTime` | End time | `End_Time` |
| `timeEntryFields.notes` | Notes | `Notes` |
| `timeEntryFields.worker` | Worker lookup / owner | `Worker` |
| `timeEntryFields.totalHours` | Calculated hours | `Total_Hours` |
| `timeEntryFields.createdByWidget` | Checkbox → triggers workflow | `Created_By_Widget` |
| `timeEntryFields.status` | `open` / `closed` | `Status` |
| `timeEntryFields.syncStatus` | Sync tracking (optional) | `Sync_Status` |
| `jobsFields.name` | Job display name | `Deal_Name` |
| `jobsFields.street` | Street address | `Street` |
| `jobsFields.city` | City | `City` |
| `jobsFields.stage` | Stage for filtering closed-lost | `Stage` |
| `jobsFields.assignedWorkers` | Crew assignment field | `Assigned_Workers` |
| `excludedStages` | Stages hidden from job picker | `["Closed Lost", ...]` |
| `expectedJobModule` | PageLoad module validation | `Deals` |

### Zoho-side setup (out of widget scope)

1. Create a **checkbox field** `Created_By_Widget` on the Time Entry module.
2. Create a **workflow** that fires when `Created_By_Widget = true` on create.
3. The workflow calls a **Deluge function** that pushes the record to Supabase.

## Architecture

- Reads and writes entirely via the **Zoho Embedded App SDK** (no Supabase client in the widget).
- On submit: `insertRecord` into Time Entry with `Created_By_Widget = true`.
- Widget placement: **detailview** on the job record (configurable in `plugin-manifest.json`).
